# RECIPE_GUIDE.md

レシピをマークダウンで書くときのルール。Claude Codeで新しいレシピを追加・編集するときはこのファイルに従うこと。

## ファイル名

- ローマ字で書く（例：`yude-tamago.md`、`karaage.md`）
- ハイフン区切り、小文字、拡張子は `.md`

## 構成

```
# 料理名

一言の説明。どんな料理か、なぜこの作り方かが伝わる程度。

---

## バリエーション名（複数ある場合）

簡潔な特徴の説明。1文。

1. 手順
2. 手順
3. 手順

**ポイント:** コツや補足。
```

- バリエーションが1つしかない場合はサブセクション不要。`# 料理名` の下に直接書く
- バリエーション間は `---` で区切る

## 文体

- 体言止めか短い文で書く（「〜する」）。丁寧語（「〜します」）は使わない
- 手順は番号つきリスト。材料リストは箇条書き
- 時間と火加減は **太字** にする（例：**7分**、**中火**）
- 余計な前置きや感想は書かない。事実だけ
- 分量が曖昧でもOK。「適量」「1cmくらい」など普段の言葉で書く

## 材料と道具

- 明らかなもの（鍋、フライパン、包丁など）は省略してよい
- 特殊な道具や意外な材料がある場合だけ「材料」「道具」セクションを設ける

## メモ・ポイント

- コツや応用は `**ポイント:**` で手順の末尾に書く
- 長くなる場合は `## メモ` セクションを最後に追加

## 出典

- 元レシピがある場合はファイル末尾に1行だけ書く
- 書き方：`参考：○○`（例：`参考：dancyu`、`参考：長谷川あかり`）
- URLは不要。名前だけでよい
- 自分で考えたレシピは出典不要

## README

- `README.md` をリポジトリルートに置き、全レシピの一覧を管理する
- カテゴリ別に分類する（例：ドリンク、だし・調味料、おかず、お菓子）
- 新しいレシピを追加したら README も更新する
- 書き方：

```markdown
## ドリンク
- [コーヒー](coffee.md)
- [基本のスムージー](smoothie.md)

## だし・調味料
- [だし](dashi.md)
- [ごまドレッシング](goma-dressing.md)
- [基本のマヨネーズ](mayonnaise.md)
- [ラー油](rayu.md)

## おかず
- [肉豆腐](niku-dofu.md)
- [肉野菜炒め](niku-yasai-itame.md)
- [なめこ汁](nameko-jiru.md)
- [卵とウィンナーの悶絶炒め](tamago-wiener-itame.md)

## お菓子
- [アーモンドクッキー](almond-cookie.md)
```

## やらないこと

- カロリーや栄養素の記載
- 写真の説明文（画像は `images/` に置くが、本文では参照しない）
- 「おいしいです！」のような感想
- 出典URL（名前だけで十分）
